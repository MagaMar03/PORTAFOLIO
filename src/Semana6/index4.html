Eventos, Renderizado y Consumo de API

¿Qué aprendí?

Aprendí que el desarrollo de aplicaciones web con un Framework JS como React permite manejar de forma eficiente eventos, como clics de botones, envío de formularios o escritura en campos de entrada, a través de funciones llamadas "manejadores de eventos" (onClick, onChange, onSubmit, etc.). También aprendí a utilizar renderizado condicional para mostrar diferentes elementos o componentes dependiendo del estado de la aplicación, usando operadores como if, ternario ( ? : ) y expresiones lógicas (&&). Del mismo modo, aprendí el renderizado iterativo, útil cuando se desea mostrar listas de elementos (por ejemplo, productos o usuarios), utilizando el método .map() para recorrer arreglos y renderizar componentes dinámicamente.

También comprendí la importancia de los formularios en las aplicaciones web, y cómo controlar los inputs a través de componentes controlados, donde el valor del campo está ligado al estado del componente (useState). Aprendí a gestionar formularios usando onChange para actualizar los datos y onSubmit para procesarlos. Otro aspecto importante fue la incorporación de rutas mediante librerías como react-router-dom, lo que permite navegar entre diferentes vistas de la aplicación sin recargar la página, manteniendo una experiencia fluida y moderna. El uso de rutas (Route, Link, useNavigate) es esencial para construir aplicaciones de varias páginas con navegación controlada desde el Front-End.

¿Cómo aprendí?

Aprendí principalmente a través de prácticas guiadas y laboratorios donde desarrollé pequeñas aplicaciones que incluían formularios, rutas dinámicas y consumo de datos desde APIs. En este proceso, entendí cómo funciona el consumo de APIs externas, es decir, cómo obtener información desde servidores externos utilizando métodos como fetch o la librería Axios. Aprendí el uso de promesas y funciones async/await, que permiten manejar peticiones asincrónicas de forma más clara y legible. Por ejemplo, al hacer una solicitud a una API para traer información del clima o una lista de usuarios, utilizaba axios.get() dentro de una función async y controlaba errores con bloques try/catch.

Durante estos ejercicios, me familiaricé con el ciclo de vida de los componentes, particularmente el uso del hook useEffect para ejecutar funciones cuando el componente se monta o cuando cambian ciertas variables. Esto es muy útil cuando se trabaja con APIs, ya que permite hacer peticiones automáticamente al iniciar un componente. También aprendí a mostrar mensajes de carga (loading...), errores y datos obtenidos de manera dinámica usando renderizado condicional, lo que mejora la experiencia del usuario. En conjunto, estas herramientas me permitieron crear aplicaciones funcionales y modernas, con interfaces que responden a eventos, muestran datos actualizados desde Internet y permiten navegar de forma dinámica sin recargar la página.
