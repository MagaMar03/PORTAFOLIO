Arquitecturas y Servicios Web

Arquitecturas de las Aplicaciones Web

Las arquitecturas de aplicaciones web definen cómo se estructuran las capas de una aplicación para facilitar su desarrollo, mantenimiento y escalabilidad. Una arquitectura bien diseñada permite una mejor distribución de responsabilidades entre cliente, servidor y servicios externos.

Arquitectura C/S Clásica - Múltiples Páginas: En este modelo, cada acción del usuario genera una nueva solicitud al servidor, que responde con una nueva página HTML. Es simple pero menos eficiente en términos de experiencia de usuario.

Arquitectura C/S Multicapa - Múltiples Páginas: Se basa en separar las responsabilidades entre presentación, lógica de negocio y acceso a datos. Cada capa interactúa con la siguiente, favoreciendo la escalabilidad.

Arquitectura C/S Multicapa – SPA: Una SPA (Single Page Application) carga una sola página HTML y actualiza dinámicamente el contenido mediante JavaScript. Esto mejora la velocidad y experiencia del usuario.

Arquitectura C/S Multicapa – Híbrida: Combina las ventajas de las SPA y las aplicaciones tradicionales (MPA), cargando páginas completas cuando es necesario y usando AJAX u otras técnicas para actualizaciones parciales.

Arquitectura Hexagonal: Enfatiza la independencia entre la lógica central de la aplicación y los servicios externos. Esto permite cambiar la base de datos, interfaces o frameworks sin afectar la lógica principal.

Servidores Web y Hosting

Un servidor web es un software que responde a las solicitudes HTTP enviadas por un navegador. Los servicios de hosting permiten almacenar y servir archivos web al público. Existen opciones como el hosting compartido, servidores dedicados y cloud hosting.

Servicio Hosting: Incluye espacio en un servidor, herramientas para gestión de archivos, bases de datos y correos. Es una opción económica para proyectos pequeños o personales.

Cloud Hosting: Utiliza servidores distribuidos que trabajan en conjunto, ofreciendo mayor disponibilidad, escalabilidad automática y redundancia frente a fallos.

Administración de Servidores Web

Administrar un servidor web implica configurar servicios, asegurar el acceso, gestionar dominios y garantizar su disponibilidad. La gestión de nombres de dominio es el proceso de vincular un dominio con la IP del servidor donde se aloja el sitio.

Gestión de dominio: Requiere comprar un dominio, asignarlo a un servidor web y configurar los registros DNS (como A, CNAME, MX).

Configuración de servidores: Implica instalar software como Apache o Nginx, configurar puertos, reglas de firewall, certificados SSL y políticas de seguridad.

Funcionamiento del Server Side

El server-side se refiere a los procesos que ocurren en el servidor, como la generación de HTML dinámico, consultas a bases de datos y validaciones. Esto permite entregar contenido personalizado y seguro al cliente.

Los lenguajes más comunes son PHP, Node.js, Python (con frameworks como Django o Flask) y Java (con JSP o Spring). Estos lenguajes permiten construir aplicaciones robustas con acceso seguro a datos y lógica de negocio.

Servidor Apache Tomcat y JSP

Tomcat es un contenedor de servlets y motor de JSP desarrollado por Apache Foundation. Permite ejecutar aplicaciones Java basadas en tecnologías como JSP y Servlets.

Con JSP (Java Server Pages), se pueden escribir páginas dinámicas mezclando código HTML y Java, facilitando el desarrollo de interfaces interactivas conectadas a bases de datos.

La configuración de Tomcat involucra definir puertos, contextos de aplicación, rutas y variables de entorno en archivos como server.xml.

Gestión de Dependencias con Maven

Maven es una herramienta de automatización de compilación y gestión de dependencias para proyectos Java. Permite incluir bibliotecas externas fácilmente y construir proyectos de forma estandarizada.

Usa un archivo pom.xml donde se declaran todas las dependencias, plugins, configuraciones de build y perfiles de ejecución.

Maven facilita la integración continua y mejora la portabilidad del proyecto entre distintos entornos de desarrollo.
