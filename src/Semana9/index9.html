Explorando la Arquitectura Web

Arquitecturas de las Aplicaciones Web

Las arquitecturas de aplicaciones web definen cómo se estructuran y comunican los componentes de una aplicación. Algunas arquitecturas comunes son:

Arquitectura C/S Clásica – Múltiples Páginas: El cliente solicita páginas y el servidor las entrega completas. Se recarga toda la página al navegar.

Arquitectura C/S Multicapa – Múltiples Páginas: Separación en capas: presentación, lógica y datos. Permite mayor escalabilidad y mantenimiento.

Arquitectura C/S Multicapa – SPA: Solo se carga una página. Los contenidos se actualizan dinámicamente vía JavaScript, sin recargar toda la web.

Arquitectura C/S Multicapa Híbrida (SPA / MPA): Combinación de múltiples páginas y zonas SPA. Usada en sitios complejos como e-commerce.

Arquitectura Hexagonal: También llamada "Ports and Adapters", separa la lógica central de las interfaces. Alta mantenibilidad.

Servidores Web / Hosting

Un servidor web entrega contenidos al navegador del usuario. Existen opciones de alojamiento como hosting tradicional, servidores virtuales o cloud hosting (Amazon, Azure, etc.), que ofrecen escalabilidad automática.

Administración de Servidores Web

Involucra registrar y administrar nombres de dominio (DNS), y configurar correctamente el servidor web (puertos, acceso, seguridad, etc.).

Funcionamiento del Server Side

En el lado servidor (server-side), se ejecuta la lógica del backend (PHP, JSP, Node.js, etc.). Se conecta con bases de datos y procesa peticiones que luego retorna al cliente.

Servidor Apache Tomcat con JSP

Apache Tomcat es un contenedor de servlets que permite ejecutar páginas JSP. Interpreta el código Java del lado servidor, generando contenido dinámico HTML.

Gestión de dependencias con Maven

Maven es una herramienta de automatización de compilación y gestión de proyectos Java. Facilita agregar librerías externas (dependencias) desde un repositorio central usando un archivo llamado pom.xml.
